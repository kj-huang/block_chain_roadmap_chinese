# 礦圈世界 — 挖礦與驗證路線圖

> 礦圈是區塊鏈基礎設施的核心 — 無論是 PoW 挖礦還是 PoS 驗證，都是維持網路安全的關鍵角色。
> 隨著以太坊在 2022 年從 PoW 轉向 PoS，「挖礦」的定義已大幅擴展為「驗證與基礎設施運營」。

## 圖例

| 標記 | 意義 |
|------|------|
| 🟠 | 個人推薦 — 強烈建議優先學習 |
| 🟡 | 建議學習 — 有助於全面理解 |
| ⬜ | 任君挑選 — 依個人興趣深入 |

## Level 1：挖礦基礎概念

### 🟠 PoW 挖礦 vs PoS 驗證

| | PoW 挖礦 | PoS 驗證 |
|---|---|---|
| 核心機制 | 用算力競爭解題權 | 用質押代幣獲得驗證權 |
| 硬體需求 | ASIC 礦機或高階 GPU | 一般伺服器即可 |
| 能耗 | 極高 | 極低 |
| 進入門檻 | 高（設備 + 電費） | 中（質押資金 + 技術能力） |
| 代表 | Bitcoin、Litecoin、Kaspa | Ethereum、Solana、Cosmos |
| 趨勢 | 逐漸集中於 Bitcoin | 成為主流共識機制 |

### 🟡 關鍵名詞

| 名詞 | 說明 |
|------|------|
| 算力 (Hashrate) | 礦機每秒可計算的雜湊次數 |
| 難度 (Difficulty) | 網路自動調整的挖礦難度，確保出塊時間穩定 |
| 出塊獎勵 (Block Reward) | 成功出塊的礦工/驗證者獲得的代幣獎勵 |
| 減半 (Halving) | Bitcoin 每 ~4 年出塊獎勵減半（2024 年已完成第四次減半） |
| 礦池 (Mining Pool) | 多個礦工合作挖礦，按貢獻分配獎勵 |
| 質押 (Staking) | 鎖定代幣參與 PoS 驗證 |
| 罰沒 (Slashing) | 驗證者作惡或離線時被沒收部分質押品 |

### 🟡 礦池 vs 單獨挖礦

| | 單獨挖礦 | 加入礦池 |
|---|---|---|
| 獎勵 | 獨得全部出塊獎勵 | 按算力比例分配 |
| 穩定性 | 極不穩定（可能數月無收入） | 穩定的小額收入 |
| 門檻 | 需要極大算力 | 低門檻即可參與 |
| 建議 | 不建議個人嘗試 | 🟠 新手推薦 |

## Level 2：PoW 挖礦

### 🟡 Bitcoin 挖礦

Bitcoin 挖礦已進入高度專業化的階段，主要由大型礦場主導。

**硬體演進**：
```
CPU → GPU → FPGA → ASIC（現階段）
```

| 設備 | 說明 |
|------|------|
| ASIC 礦機 | 專為 SHA-256 演算法設計的晶片，如 Antminer S21 |
| 電源供應器 | 礦機功耗極高（單台可達 3000W+） |
| 散熱設備 | 礦機產生大量熱能，需要通風或水冷 |

**關鍵考量**：
- 電費是最大的營運成本（通常佔 60-80%）
- 台灣電費相對較高，個人挖 Bitcoin 通常不划算
- 需考量礦機折舊和二手市場價格
- 減半後收益下降，需要更低的電力成本才能維持獲利

### 🟡 主流礦池

| 礦池 | 說明 |
|------|------|
| [F2Pool](https://www.f2pool.com/) | 老牌礦池，支援多幣種 |
| [Antpool](https://www.antpool.com/) | 比特大陸旗下 |
| [ViaBTC](https://www.viabtc.com/) | 支援多種分配模式 |
| [Foundry](https://foundrydigital.com/) | 北美最大礦池 |
| [Braiins Pool](https://braiins.com/pool) | 原 Slush Pool，最早的礦池 |

### ⬜ 其他 PoW 幣種

| 幣種 | 演算法 | 特色 |
|------|--------|------|
| Litecoin (LTC) | Scrypt | Bitcoin 的「白銀」，合併挖礦 DOGE |
| Kaspa (KAS) | kHeavyHash | 新興 PoW 幣，DAG 架構，高 TPS |
| Monero (XMR) | RandomX | 隱私幣，CPU 挖礦友善 |
| Zcash (ZEC) | Equihash | 隱私幣，可選擇性透明 |

### 🟠 挖礦收益計算

在投入資金前，務必先做收益計算：

| 項目 | 說明 |
|------|------|
| 算力 | 你的礦機/礦場的總算力 |
| 電費 | 每度電多少錢？每月總電費？ |
| 礦機成本 | 購入價格、預估使用壽命 |
| 幣價 | 當前幣價和你的價格預期 |
| 難度趨勢 | 全網算力是否持續成長？ |
| 礦池手續費 | 通常 1-3% |

工具推薦：
- [WhatToMine](https://whattomine.com/) — 多幣種挖礦收益計算器
- [Braiins Insights](https://insights.braiins.com/) — Bitcoin 挖礦數據

## Level 3：PoS 驗證與質押

### 🟠 以太坊驗證者

以太坊從 2022 年 9 月起完全轉為 PoS，任何人都可以成為驗證者。

**基本要求**：

| 項目 | 需求 |
|------|------|
| 質押金額 | 32 ETH（每個驗證者節點） |
| 硬體 | 4 核 CPU、16GB RAM、2TB SSD（建議 NVMe） |
| 網路 | 穩定的寬頻連線（上下行 25Mbps+） |
| 軟體 | 執行層客戶端 + 共識層客戶端 |
| 可用性 | 需 24/7 運行，離線會被輕微罰沒 |

**執行層客戶端**（擇一）：

| 客戶端 | 語言 | 說明 |
|--------|------|------|
| [Geth](https://geth.ethereum.org/) | Go | 最多人使用 |
| [Nethermind](https://nethermind.io/) | C# | 效能優異 |
| [Besu](https://besu.hyperledger.org/) | Java | 企業友善 |
| [Erigon](https://github.com/ledgerwatch/erigon) | Go | 磁碟使用最少 |

**共識層客戶端**（擇一）：

| 客戶端 | 語言 | 說明 |
|--------|------|------|
| [Prysm](https://prysmaticlabs.com/) | Go | 社群最大 |
| [Lighthouse](https://lighthouse.sigmaprime.io/) | Rust | 效能穩定 |
| [Teku](https://consensys.io/teku) | Java | ConsenSys 維護 |
| [Nimbus](https://nimbus.team/) | Nim | 資源佔用最少 |
| [Lodestar](https://lodestar.chainsafe.io/) | TypeScript | 客戶端多樣性 |

> **建議**：選擇非主流客戶端組合（如 Nethermind + Lighthouse），有助於網路的客戶端多樣性。

### 🟠 流動性質押（降低門檻）

沒有 32 ETH 也可以參與以太坊質押。

| 協議 | 說明 | 最低門檻 |
|------|------|----------|
| [Lido](https://lido.fi/) | 市佔最高，獲得 stETH 可在 DeFi 中使用 | 任意金額 |
| [Rocket Pool](https://rocketpool.net/) | 更去中心化，獲得 rETH | 0.01 ETH |
| [Frax Ether](https://frax.finance/) | 獲得 sfrxETH | 任意金額 |

**再質押 (Restaking)**：
- [EigenLayer](https://www.eigenlayer.xyz/) — 用已質押的 ETH 同時保護其他協議
- 可獲得額外收益，但也承擔額外風險

### 🟡 Cosmos 驗證者

| 項目 | 需求 |
|------|------|
| 質押 | 需進入活躍驗證者集合（競爭質押量） |
| 硬體 | 4 核 CPU、32GB RAM、500GB+ NVMe SSD |
| 佣金 | 自行設定佣金比例（通常 5-10%） |
| 收益 | 來自區塊獎勵 + 交易手續費 + 委託者佣金 |

### 🟡 Solana 驗證者

| 項目 | 需求 |
|------|------|
| 質押 | 自質押 + 吸引委託質押 |
| 硬體 | 12 核 CPU、256GB RAM、高速 NVMe（門檻較高） |
| 網路 | 300Mbps+ 頻寬 |
| 成本 | 硬體成本和投票手續費較高 |

### 🟡 委託質押 (Delegated Staking)

不想自己跑驗證者？可以將代幣委託給信任的驗證者。

- 幾乎所有 PoS 鏈都支援委託質押
- 驗證者收取佣金（通常 5-15%），其餘分給委託者
- 選擇標準：正常運行時間、佣金比例、社群貢獻、治理參與

## Level 4：節點運營

### 🟡 節點類型

| 類型 | 說明 | 磁碟需求 |
|------|------|----------|
| 全節點 (Full Node) | 驗證所有交易，保存最近狀態 | ~1TB |
| 輕節點 (Light Node) | 只驗證區塊頭，依賴全節點 | 很小 |
| 歸檔節點 (Archive Node) | 保存所有歷史狀態 | ~15TB+ |

### 🟡 以太坊節點架設

**步驟概覽**：

1. **選擇硬體** — 自建伺服器或雲端 VPS
2. **安裝執行層客戶端** — 如 Geth，同步區塊鏈資料
3. **安裝共識層客戶端** — 如 Lighthouse，處理 PoS 共識
4. **設定防火牆** — 開放必要的 P2P 通訊埠
5. **監控** — 使用 Grafana + Prometheus 監控節點狀態
6. **維護** — 定期更新客戶端、處理磁碟空間

**推薦工具**：
- [DAppNode](https://dappnode.io/) — 一鍵部署節點的硬體/軟體方案
- [Stereum](https://stereum.net/) — 以太坊節點管理工具
- [eth-docker](https://eth-docker.net/) — Docker 化的以太坊節點部署

### ⬜ RPC 節點服務營運

如果你的節點穩定運行，可以考慮提供 RPC 服務：

| 平台 | 說明 |
|------|------|
| [Pocket Network](https://pokt.network/) | 去中心化 RPC 基礎設施 |
| [Lava Network](https://lavanet.xyz/) | 去中心化 RPC 聚合 |
| 自建 RPC | 提供付費 API，服務 DApp 開發者 |

## Level 5：經濟分析與風險

### 🟠 挖礦/驗證收益模型

**PoW 挖礦收益**：
```
每日收益 = (你的算力 / 全網算力) × 每日出塊獎勵 × 幣價
每日成本 = 電費 + 場地 + 人力 + 設備折舊
每日利潤 = 每日收益 - 每日成本
```

**PoS 驗證收益**：
```
年化收益率 ≈ 基礎獎勵 + 交易小費 + MEV 收益
以太坊驗證者年化 ≈ 3-5%（含 MEV-Boost）
```

### 🟡 投資回報期 (ROI)

| 因素 | 影響 |
|------|------|
| 幣價走勢 | 幣價上漲加速回本，下跌延長甚至虧損 |
| 難度變化 | PoW 全網算力增加會降低個人收益 |
| 電費波動 | 電力成本直接影響利潤率 |
| 硬體殘值 | 礦機淘汰後的二手價值 |
| 減半事件 | Bitcoin 減半直接將出塊獎勵腰斬 |

### 🟡 罰沒風險 (Slashing)

PoS 驗證者面臨的主要風險：

| 違規行為 | 處罰 |
|----------|------|
| 雙重投票 (Double Voting) | 罰沒至少 1 ETH |
| 環繞投票 (Surround Voting) | 罰沒至少 1 ETH |
| 長時間離線 | 輕微罰款（非 slashing），但會持續減少餘額 |
| 關聯性罰沒 | 同時期越多驗證者被罰沒，你的罰沒金額越高 |

**防範措施**：
- 使用 UPS（不斷電系統）避免停電離線
- 不要同時在多台機器上運行同一把驗證者金鑰
- 定期備份驗證者金鑰的 Slashing Protection 資料庫
- 使用遠端簽名器 (Remote Signer) 如 Web3Signer

### ⬜ 電力成本與地理選址

全球主要挖礦地點：

| 地區 | 優勢 |
|------|------|
| 德克薩斯 (美國) | 電力市場自由化，可再生能源豐富 |
| 冰島 | 地熱發電，天然散熱 |
| 哈薩克 | 電價低廉 |
| 巴拉圭 | 水力發電，電費極低 |
| 北歐 | 可再生能源 + 涼爽氣候 |

### 🟡 Bitcoin 減半週期

| 次數 | 年份 | 出塊獎勵 |
|------|------|----------|
| 創世 | 2009 | 50 BTC |
| 第一次 | 2012 | 25 BTC |
| 第二次 | 2016 | 12.5 BTC |
| 第三次 | 2020 | 6.25 BTC |
| 第四次 | 2024 | 3.125 BTC |
| 第五次 | ~2028 | 1.5625 BTC |

歷史上，減半後 12-18 個月通常伴隨價格上漲，但過去表現不代表未來。

## Level 6：進階基礎設施

### ⬜ MEV 提取

| 概念 | 說明 |
|------|------|
| MEV (最大可提取價值) | 透過交易排序獲取的額外價值 |
| [Flashbots](https://flashbots.net/) | 透明化 MEV 提取的生態系 |
| [MEV-Boost](https://boost.flashbots.net/) | 驗證者可委託專業 Builder 建構區塊，獲得 MEV 分潤 |
| PBS | 提議者-建構者分離，以太坊的長期 MEV 方案 |

**對驗證者的影響**：
- 啟用 MEV-Boost 可額外增加 ~1-2% 年化收益
- 幾乎所有以太坊驗證者都應啟用 MEV-Boost

### ⬜ DePIN (去中心化實體基礎設施網路)

新興領域 — 用代幣激勵人們提供實體世界的基礎設施。

| 專案 | 類型 | 說明 |
|------|------|------|
| Helium | 無線網路 | 提供 LoRaWAN / 5G 熱點，賺取 HNT |
| Hivemapper | 地圖 | 行車記錄器收集街景數據 |
| Render | GPU 運算 | 提供閒置 GPU 給 3D 渲染需求 |
| io.net | GPU 運算 | 去中心化 GPU 算力網路（AI 訓練） |

### ⬜ 分散式儲存挖礦

| 專案 | 說明 |
|------|------|
| [Filecoin](https://filecoin.io/) | 提供硬碟空間賺取 FIL，需質押 + 特殊硬體 |
| [Arweave](https://arweave.org/) | 永久儲存資料，挖礦需儲存歷史資料 |
| [Chia](https://chia.net/) | 空間證明 (Proof of Space)，用硬碟空間挖礦 |

## 建議學習順序

```
理解 PoW 與 PoS 的差異
        │
        ▼
  選擇一條入門路線
   ┌─────┴─────┐
   ▼           ▼
 PoW 挖礦    PoS 質押
 (投資較高)   (門檻較低)
   │           │
   ▼           ▼
收益計算     委託質押
成本分析     (最簡單的開始)
   │           │
   ▼           ▼
 礦池選擇   自建驗證者
 礦機採購   節點運營
   │           │
   └─────┬─────┘
         ▼
   進階：MEV、DePIN
```
