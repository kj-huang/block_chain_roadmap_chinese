## 一、區塊鏈基本名詞

這些是你在區塊鏈世界裡每天都會遇到的詞彙，務必搞懂。

### 核心概念

| 名詞 | 說明 |
|------|------|
| 區塊 (Block) | 一批交易紀錄的集合，按順序串接成鏈 |
| 鏈 (Chain) | 區塊依照時間順序串聯而成的資料結構 |
| 節點 (Node) | 網路中參與驗證和傳播交易的電腦 |
| 哈希函數 (Hash Function) | 將任意資料轉換成固定長度摘要的單向函數（如 SHA-256） |
| Merkle Tree | 用於高效驗證大量資料完整性的樹狀結構 |
| 公鑰 / 私鑰 | 非對稱加密的金鑰對；公鑰可公開，私鑰必須保密 |
| 數位簽章 | 用私鑰對交易簽名，任何人可用公鑰驗證真偽 |
| 錢包地址 | 由公鑰衍生的識別碼，用來接收加密貨幣 |
| 助記詞 (Mnemonic) | 12 或 24 個英文單字，用來備份和還原私鑰 |
| Gas | 在以太坊上執行運算所需支付的計算單位 |
| 手續費 (Transaction Fee) | 支付給礦工或驗證者的費用，確保交易被處理 |

### 進階概念

| 名詞 | 說明 |
|------|------|
| 智能合約 (Smart Contract) | 部署在區塊鏈上、自動執行的程式碼 |
| ERC-20 | 以太坊上同質化代幣的標準（如 USDT、UNI） |
| ERC-721 | 非同質化代幣 (NFT) 的標準 |
| ERC-1155 | 可同時處理同質化與非同質化代幣的多重標準 |
| Layer 1 (L1) | 底層主鏈，如 Ethereum、Bitcoin、Solana |
| Layer 2 (L2) | 建立在 L1 之上的擴展方案，如 Arbitrum、Optimism |
| DApp | 去中心化應用程式，前端連接智能合約 |
| TVL (Total Value Locked) | 鎖定在 DeFi 協議中的總資產價值 |

## 二、共識機制

共識機制是區塊鏈的靈魂 — 它決定了網路中的節點如何就「哪些交易是合法的」達成一致。

### 🟠 工作量證明 (Proof of Work, PoW)

- 透過算力競爭來產生新區塊
- 最早且最經典的共識機制，Bitcoin 使用至今
- 優點：安全性極高、去中心化程度高
- 缺點：耗能巨大、交易速度慢
- 代表：Bitcoin、Litecoin、Kaspa

### 🟠 權益證明 (Proof of Stake, PoS)

- 依照持有代幣數量和時間來決定驗證資格
- 以太坊於 2022 年 9 月「合併」(The Merge) 後正式轉為 PoS
- 優點：節能（比 PoW 減少 ~99.95% 能耗）、可擴展性較佳
- 缺點：可能導致「富者越富」
- 代表：Ethereum、Cardano、Polkadot

### 🟡 其他共識機制

| 機制 | 說明 | 代表 |
|------|------|------|
| DPoS (委託權益證明) | 持幣者投票選出代表來驗證交易 | EOS、Tron |
| PoA (權威證明) | 由預先授權的節點驗證，適合私有鏈 | VeChain |
| PBFT (實用拜占庭容錯) | 節點間多輪投票達成共識 | Hyperledger Fabric |
| PoH (歷史證明) | 用可驗證的時間戳排序交易 | Solana |
| Tendermint / CometBFT | 結合 PoS 和 BFT 的共識引擎 | Cosmos 生態系 |

### 🟡 拜占庭將軍問題

這是分散式系統的經典問題：在部分節點可能作惡的情況下，如何讓所有誠實節點達成一致？所有共識機制本質上都是在解決這個問題。

## 三、記帳與資料結構

### 🟠 兩大記帳模型

| | UTXO 模型 (Bitcoin) | Account 模型 (Ethereum) |
|---|---|---|
| 概念 | 每筆交易消耗舊的「未花費輸出」，產生新的輸出 | 類似銀行帳戶，直接記錄餘額 |
| 隱私性 | 較高（每次交易可用新地址） | 較低（地址固定） |
| 智能合約 | 不易實作複雜邏輯 | 天然支援 |
| 代表 | Bitcoin、Litecoin | Ethereum、Solana |

### 🟡 分散式帳本 (Distributed Ledger)

- 帳本不存在單一中心，而是由網路中每個節點各自保存一份副本
- 任何人都可以驗證帳本的正確性
- 一旦寫入就幾乎不可能竄改

### 🟡 區塊結構

```
區塊 (Block)
├── 區塊頭 (Header)
│   ├── 前一區塊的雜湊值 (Previous Hash)
│   ├── 時間戳 (Timestamp)
│   ├── Merkle Root（所有交易的摘要）
│   ├── 難度目標 (Difficulty Target)
│   └── Nonce（PoW 中礦工嘗試的隨機數）
└── 區塊體 (Body)
    └── 交易清單 (Transaction List)
```

## 四、P2P 技術

區塊鏈之所以能「去中心化」，關鍵在於點對點 (Peer-to-Peer) 網路架構。

### 🟠 對等式網路基礎

- 沒有中央伺服器，每個節點既是客戶端也是伺服器
- 節點可自由加入或離開網路
- 資料在節點之間直接傳播

### 🟡 關鍵技術

| 技術 | 說明 |
|------|------|
| Gossip Protocol | 節點像「八卦」一樣隨機向鄰居傳播資訊，最終全網同步 |
| DHT (分散式雜湊表) | 在沒有中心索引的情況下，高效定位網路中的資料 |
| NAT 穿越 | 讓防火牆後的節點也能互相連線 |

### ⬜ libp2p

- 由 Protocol Labs（IPFS 開發團隊）打造的模組化 P2P 網路框架
- 被 Ethereum 2.0、Polkadot、Filecoin 等專案採用
- 提供傳輸、身份驗證、路由、內容發現等功能

## 五、商業知識

技術之外，你也需要理解區塊鏈世界的商業邏輯。

### 🟠 代幣經濟學 (Tokenomics)

- **供給**：總量、流通量、通膨/通縮機制（如 Bitcoin 的 2100 萬上限、ETH 的 EIP-1559 銷毀機制）
- **分配**：團隊、投資者、社群、生態基金各佔多少比例？
- **解鎖時程 (Vesting)**：代幣何時釋放？會不會造成拋壓？
- **實用性 (Utility)**：代幣在生態系中的用途是什麼？

### 🟡 DAO (去中心化自治組織)

- 由智能合約治理的組織，成員透過代幣投票決策
- 範例：MakerDAO（穩定幣治理）、Uniswap（協議升級投票）
- 挑戰：投票率低、治理攻擊、法律地位不明確

### 🟡 DeFi 基礎概念

| 類型 | 說明 | 範例 |
|------|------|------|
| DEX (去中心化交易所) | 無需中介即可交易代幣 | Uniswap、Raydium |
| 借貸 (Lending) | 存入資產賺利息、借出資產付利息 | Aave、Compound |
| 穩定幣 (Stablecoin) | 錨定法幣價值的加密貨幣 | USDT、USDC、DAI |
| 收益聚合 (Yield Aggregator) | 自動尋找最佳收益策略 | Yearn Finance |

### 🟡 監管環境概覽

| 地區 | 現況 |
|------|------|
| 台灣 | 金管會將虛擬資產列為管理範疇，要求業者遵循洗錢防制規範 |
| 美國 | SEC 積極監管，多起訴訟案件；ETF 已獲批准 |
| 歐盟 | MiCA (加密資產市場監管框架) 已正式實施 |
| 日本 | 較早立法監管，持牌交易所制度成熟 |

### ⬜ Web3 vs Web2 商業模式差異

| | Web2 | Web3 |
|---|---|---|
| 資料所有權 | 平台擁有 | 用戶擁有 |
| 價值捕獲 | 廣告、訂閱 | 代幣經濟、協議費用 |
| 信任模型 | 信任中介 | 信任程式碼 (Trustless) |
| 身份系統 | 帳號密碼、OAuth | 錢包地址、DID |
| 開源程度 | 封閉為主 | 開源為主（合約可驗證） |
